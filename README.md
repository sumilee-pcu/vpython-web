# vpython-web

**Web VPython powered by real CPython** — 브라우저에서 진짜 CPython(Pyodide)으로 VPython 물리 시뮬레이션을 실행하는 웹 환경.

[GlowScript](https://www.glowscript.org/)와 달리 Python→JS 트랜스파일이 아니라 WebAssembly로 컴파일된 실제 CPython을 사용하므로, 장기적으로 numpy 등 표준 파이썬 생태계를 그대로 쓸 수 있는 것이 목표.

웹 데모: [https://sumilee-pcu.github.io/vpython-web/](https://sumilee-pcu.github.io/vpython-web/)

## 교육적 필요성

물리와 프로그래밍은 모두 “보이지 않는 관계를 모델로 세우고, 결과를 확인하며, 다시 수정하는” 학습 과정이다. 하지만 학교 현장에서는 파이썬 설치, 패키지 설치, 그래픽 환경 설정, 학생별 기기 차이 때문에 수업 시간이 환경 구성에 많이 쓰인다. vpython-web은 브라우저만 열면 곧바로 3D 물리 시뮬레이션을 작성하고 실행할 수 있게 하여, 수업의 초점을 설치가 아니라 개념 탐구와 코드 수정에 둔다.

이 프로젝트는 특히 다음 문제를 줄이기 위해 만들어졌다.

- **설치 장벽**: 별도 파이썬/VPython/IDE 설치 없이 웹에서 실행한다.
- **개념과 코드의 분리**: 공의 위치, 속도, 힘, 궤도, 충돌을 코드와 3D 장면으로 동시에 확인한다.
- **수업 재현성**: 같은 URL과 같은 예제를 쓰면 교사와 학생이 동일한 출발점에서 실험한다.
- **탐구형 수업**: 학생이 수치 하나를 바꾸고 즉시 결과를 확인하면서 가설을 검증한다.
- **파이썬 생태계 연결**: 장기적으로 실제 CPython 기반의 numpy/scipy/sympy 활용까지 확장할 수 있다.

## 교육 효과

- **물리 개념의 시각화**: 위치, 속도, 가속도, 중력, 충돌, 진동, 파동처럼 추상적인 개념을 움직이는 3D 객체로 관찰한다.
- **수식과 코드의 연결**: `F=ma`, 등가속도 운동, 원운동, 조화진동 같은 식이 코드 한 줄과 화면의 운동으로 이어지는 과정을 경험한다.
- **컴퓨팅 사고력 강화**: 변수를 정의하고, 반복문으로 시간을 진행시키고, 조건문으로 충돌과 경계를 처리하며 모델링 과정을 익힌다.
- **즉각적인 피드백**: 코드를 바꾼 뒤 Run을 누르면 결과가 바로 나타나므로 시행착오 기반 학습이 쉽다.
- **교사 자료 제작 효율**: 예제 라이브러리를 바탕으로 수업용 데모, 과제, 실험 조건을 빠르게 만들 수 있다.
- **학생 주도 탐구**: 질량, 초기 속도, 중력, 탄성 계수 등을 바꿔 보며 “왜 이렇게 움직이는가”를 스스로 질문하게 한다.

## 수업 활용 예

- 중학교/고등학교 물리: 등가속도 운동, 포물선 운동, 충돌, 원운동, 진동, 파동 단원
- 대학 기초물리/공학물리: 수치적 시간 적분, 중력 궤도, 결합 진동자, 장(field) 시각화 입문
- 정보/융합 수업: 파이썬 반복문, 조건문, 벡터, 시뮬레이션 모델링
- 프로젝트 수업: 학생이 예제를 변형해 자신만의 물리 애니메이션 제작

## 현재 예제 라이브러리

현재 웹 UI에는 30개의 물리 예제가 들어 있다. 예제는 카테고리, 난이도, 미리보기와 함께 제공되며, 선택 후 `Load`를 누르면 코드가 에디터에 로드된다.

- Mechanics: 포물선 운동, 충돌, 빗면, 기체 입자 등
- Oscillation: 단순 조화 운동, 감쇠 진동, 강제 진동, 스프링-질량, 진자 등
- Rotation: 원운동, 구심력, 회전 막대
- Gravity: 행성 궤도, 쌍성계, 케플러 법칙 느낌의 예제
- Waves: 진행파, 정상파, 횡파 줄, 격자 진동
- Fields: 전기 쌍극자, 자기장 격자, 전하 반발

## 학령별 권장 활용

예제에는 권장 학령 메타데이터가 붙어 있으며, 웹 UI의 학령 필터로 골라 볼 수 있다. 학령 구분은 엄격한 제한이 아니라 “교사가 별도 설명 없이 도입하기 쉬운 최소 수준”을 기준으로 한다.

| 권장 학령 | 활용 방향 | 예제 성격 |
|---|---|---|
| 초등 고학년 | 움직임 관찰, 3D 공간 감각, 코드를 바꾸면 결과가 달라진다는 경험 | 튀는 공, 굴러가는 공, 회전 막대, 미니 태양계, 좌표축 |
| 중학교 | 속력, 힘, 파동, 전기·자기 개념의 직관적 도입 | 빗면, 원운동, 기체 입자, 진행파, 횡파 줄, 자기장 격자 |
| 고등학교 | 수식 기반 물리 개념과 코드 모델의 연결 | 포물선 운동, 충돌, 조화진동, 구심력, 행성 궤도, 전기장, 전하 반발 |
| 대학 입문 | 수치적 모델링, 결합계, 비선형/다물체 운동의 맛보기 | 강제 진동, 결합 진동자, 이중 진자, 삼체 운동, 격자 진동 |

## 아키텍처

```
┌─────────────────────────── 브라우저 ───────────────────────────┐
│                                                                │
│  CodeMirror 에디터                                              │
│       │ (사용자 VPython 코드, rate() → await rate() 변환)        │
│       ▼                                                        │
│  Pyodide (CPython/WASM)                                        │
│    └─ src/vpython.py : VPython API 계층 (vector, box, sphere…) │
│            │ (JS 브리지: globalThis.vpw)                        │
│            ▼                                                   │
│  src/renderer.js : three.js/WebGL 렌더러 (GPU)                  │
└────────────────────────────────────────────────────────────────┘
```

- **실행 모델**: VPython의 `while True: rate(100)` 무한 루프는 실행 전에 `await rate(100)`으로 자동 변환되어, 매 프레임 브라우저에 제어권을 돌려준다. Run을 다시 누르면 세대 카운터가 올라가 이전 루프는 다음 `rate()`에서 종료된다.
- 서버 없음: 전부 정적 파일이라 GitHub Pages 등에 그대로 배포 가능.

## 실행 방법

Pyodide가 `vpython.py`를 fetch로 불러오기 때문에 로컬 웹서버가 필요하다 (`file://`로는 안 됨):

```bash
cd vpython-web
python -m http.server 8000
# 브라우저에서 http://localhost:8000 접속
```

첫 로딩 시 Pyodide(약 10MB) 다운로드로 수십 초 걸릴 수 있다 (이후 브라우저 캐시).

## 현재 지원하는 API

- `vector` (연산자, `mag`, `mag2`, `norm()`, `dot()`, `cross()`, 성분 수정 시 화면 자동 반영)
- `box`, `sphere` — pos, **axis/up (회전)**, size/radius, **length/height/width**,
  color, opacity, visible + velocity 등 자유 속성
- `cylinder` — pos(시작점), axis, radius, length 별칭, color, opacity, visible
- `color` 상수 (red, green, blue, yellow, orange, cyan, magenta, purple, white, black, gray())
- `rate(n)`, `print()` (하단 출력창), 에러 시 줄번호 표시 + 해당 줄 하이라이트
- 예제 라이브러리 드롭다운 (`examples/`의 물리 예제 30개 + 미리보기 로드)
- 마우스 드래그로 카메라 회전/줌 (OrbitControls)

### 알려진 제약

- 사용자 정의 함수 안에서 `rate()` 호출 불가 (top-level 루프에서만) — 함수 async화 미지원
- `rate()` 없는 무한루프는 탭을 멈춤 (Stop 버튼/Worker는 Phase 7)
- `scene` 객체는 스텁 (속성 설정은 조용히 무시됨 — Phase 4에서 구현)
- 백그라운드 탭에서는 브라우저 타이머 스로틀로 시뮬레이션이 느려짐 (웹 표준 동작)
